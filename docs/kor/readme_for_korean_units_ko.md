# 전통 단위 환산 (korean_units_ko) — 난이도 설계 문서

- 생성기: `generation/korean_units_ko.py`
- 평가기: `evaluation/evaluators/korean_units_ko.py`
- 데이터: `data/json/korean_units_ko_{easy,medium,hard}.jsonl`
- 태스크명: `korean_units_ko`, `korean_units_ko_easy`, `korean_units_ko_medium`, `korean_units_ko_hard`

## 이 게임은 무엇인가요? (자세한 설명)

우리 조상들은 길이를 잴 때 자·치·푼, 곡식의 부피를 잴 때 되·말·섬, 무게를 잴 때 돈·냥·근·관,
논밭의 넓이를 잴 때 평·단·정보 같은 **전통 도량형**을 썼습니다. 이 퍼즐은 그런 옛 단위로 적힌
양을 **하나의 기준 단위로 환산해 총합을 구하는** 문제입니다. 예를 들어 "3섬 4말 5되는 모두 몇
홉인가?"처럼요.

가장 중요한 약속이 하나 있습니다. **푸는 사람이 실제 전통 단위 지식을 외우고 있을 필요가
없다**는 것입니다. 문제마다 그 단위들 사이의 관계를 적은 **환산표가 함께 주어지고, 오직 그 표만
사용해서** 계산해야 합니다. 즉 "1되 = 10홉, 1말 = 10되, 1섬 = 10말" 같은 표를 보고, 이를 차례로
적용해 모든 양을 기준 단위(여기서는 홉)로 바꾼 뒤 더하면 됩니다. 그래서 이 퍼즐은 옛 단위에 대한
'암기'가 아니라, 주어진 규칙을 **여러 단계에 걸쳐 정확히 곱하고 더하는 산술 추론**을 측정합니다.

진짜 어려움은 단위 사이의 비율이 항상 10배인 것은 아니라는 데서 옵니다. 부피(되·말·섬)나
길이(치·자·장)는 모두 10배씩이라 사실상 자릿수만 맞추면 되지만, **무게의 1근 = 16냥**이나
**넓이의 1단 = 300평**처럼 10이 아닌 비율이 끼어들면 이야기가 달라집니다. 예컨대 "2근 5냥 3돈은
몇 돈?"을 풀려면 1근 = 16냥 = 160돈임을 표에서 연쇄로 끌어내, 2×160 + 5×10 + 3 = 373돈을
계산해야 합니다. 여기에 여러 사람이 가진 양을 모두 합치는 문제까지 더해지면, 큰 수를 여러 번
곱하고 더하는 동안 한 번이라도 자리를 틀리면 곧장 오답이 됩니다.

요약하면, 한복 입은 옛 시장에서 곡식과 천과 고기를 사고팔며 셈하던 상황을 떠올리면 됩니다.
규칙(환산표)은 친절하게 다 적혀 있지만, 그것을 **빠짐없이, 정확히, 여러 단계에 걸쳐 적용하는
끈기와 정밀함**이 이 게임의 승부처입니다.

## 문제 개념
- 문제에 **환산표**(예: `1냥=10돈`, `1근=16냥`, `1관=1000돈`)를 함께 제시 → 자기완결형.
- 주어진 양(예: `2근 5냥 3돈`, 또는 여러 사람의 양)을 **기준 단위로 환산해 총합**.
- **정답 형식**: `N<기준단위>` (예: `373돈`) — 정수.
- **정답 산출**: 각 항을 표의 환산값으로 곱해 더한 결정적 유일해.

## 난이도는 어떻게 조정되나요? (자세한 설명)

난이도를 가르는 손잡이는 네 가지입니다 — **① 단위 항의 개수, ② 환산 비율이 10인가 아닌가
(근=16, 단=300 같은 비-십진 비율), ③ 수의 크기, ④ 합산할 수량(사람)의 수**입니다. 모두
"얼마나 많은 곱셈·덧셈을, 얼마나 까다로운 비율로 해야 하는가"를 키우는 방향입니다.

각 난이도는 이렇게 구성됩니다. **easy**는 ×10 환산만 쓰는 부피·길이·무게(냥·돈)에서 한 수량을
2~3개 단위로 묻습니다(예: `5되 3홉 → 53홉`). 사실상 자릿수만 맞추면 되는 1~2단계 산술입니다.
**medium**은 비율이 10이 아닌 시스템(무게의 근=16냥, 넓이의 단=300평)으로 한정해, 한 수량을
2~4개 단위로 묻습니다(예: `17정보 9단 2평 → 53702평`). 비-십진 곱셈이 들어가 실수 여지가
커집니다. **hard**는 무게(근=16냥·관=1000돈)에서 **세 사람의 양을 각각 환산해 모두 더하는**
문제로, 큰 수의 다단계 곱셈·덧셈이 길게 이어집니다.

코드에서 이 손잡이는 `generate_puzzle`의 난이도 분기에 들어 있습니다.

```python
# generation/korean_units_ko.py — 난이도별 (시스템, 허용 단위, 수량 수, 최대 개수)
# easy  : ['부피','길이','무게'] / ×10 레벨만           / n_quant=1 / cmax=9
# medium: ['무게','넓이']        / 전체 레벨(근16·단300) / n_quant=1 / cmax=20  (어색한 단위 필수)
# hard  : '무게'                 / 돈·냥·근·관          / n_quant=3 / cmax=30
# 공통  : answer_cap = num//3 (최빈값 찍기 방지), 단위 항 ≥ 2
```

난이도를 조절하려면 위 네 값을 바꾸면 됩니다. 더 어렵게 하려면 `n_quant`(합산할 사람 수)와
`cmax`(개수 상한)를 키우거나 항 수를 늘리고, 더 쉽게 하려면 반대로 줄이거나 ×10 시스템만 쓰면
됩니다.

> ⚠️ **모델별 특성**: 이 퍼즐은 본질이 다단계 산술이라 **frontier 모델(Gemini-3-flash 등)은
> 거의 100%로 포화**됩니다 — 5명 합산·정답 10만 돈대까지 키워도 100%였습니다(CoT로 산술을 잘
> 함). 따라서 강한 모델로는 난이도가 분리되지 않습니다. 반면 이 프로젝트의 실제 타깃인 **작은
> 모델(Qwen 0.6B~8B 등)은 다단계 산술에서 빠르게 무너지므로** medium·hard에서 정답률이 뚜렷이
> 떨어질 것으로 기대됩니다. 즉 **작은 모델용 산술 추론 퍼즐**로 설계·활용하는 것이 적절합니다.
