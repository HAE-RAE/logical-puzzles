# 끝말잇기 최장 사슬 (word_chain_ko) — 난이도 설계 문서

- 생성기: `generation/word_chain_ko.py`
- 평가기: `evaluation/evaluators/word_chain_ko.py`
- 데이터: `data/json/word_chain_ko_{easy,medium,hard}.jsonl`
- 태스크명: `word_chain_ko`, `word_chain_ko_easy`, `word_chain_ko_medium`, `word_chain_ko_hard`

## 이 게임은 무엇인가요? (자세한 설명)

누구나 한 번쯤 해본 **끝말잇기**입니다. 앞 단어의 마지막 글자로 시작하는 단어를 계속 이어
붙이는 한국 고유의 말놀이죠(예: 사과 → 과자 → 자두 → 두부 → …). 이 퍼즐은 그 끝말잇기를
조금 비틀어, **주어진 단어 목록만으로 만들 수 있는 가장 긴 사슬이 단어 몇 개짜리인지**를
맞히는 문제입니다. 같은 단어는 한 번만 쓸 수 있습니다.

규칙은 간단하지만 답을 찾는 일은 만만치 않습니다. 단어 16개가 주어졌다고 해봅시다. 어떤
단어로 시작하느냐, 갈림길에서 어떤 단어를 고르느냐에 따라 사슬의 길이가 천차만별이 됩니다.
어떤 길로 가면 5개에서 막히고, 다른 길로 가면 9개까지 이어집니다. **가장 긴 길 하나를 찾아내야**
하는데, 가능한 순서의 가짓수가 단어 수가 늘수록 폭발적으로 많아져서 일일이 따져보기가
어렵습니다. 그래서 사람이든 AI든 **그럴듯하게 긴 사슬은 쉽게 찾지만, 진짜 '최장'인지를 확신하기는
어렵습니다.** 하나만 더 이으면 되는 길을 놓치거나, 반대로 안 이어지는 단어를 잘못 이어 길이를
부풀리기 쉽습니다.

중요한 점은, 이 퍼즐이 **어휘력(단어를 아는가)을 묻지 않는다**는 것입니다. 쓸 수 있는 단어가
문제에 모두 적혀 있으니까요. 대신 그 단어들이 만드는 **연결 관계 속에서 가장 긴 경로를 찾는
'탐색' 능력**을 측정합니다. 이는 컴퓨터과학에서 '최장 단순경로(longest simple path)'라 부르는
어려운(NP-hard) 문제와 같습니다 — 적당한 답은 금방 나오지만 최적(최대)을 보장하기는 매우 어려운,
바로 그런 종류의 문제입니다.

## 문제 개념
- 단어 목록을 문제에 **제시**(자기완결, 어휘 암기 불필요).
- 규칙: 앞 단어 끝 글자 = 다음 단어 첫 글자, 각 단어 1회.
- **정답 형식**: `N개` (최장 사슬의 단어 수, 정수).
- **정답 산출**: 메모이즈 비트마스크 DFS로 **최장 단순경로**를 정확히 계산(유일한 최댓값).

## 난이도는 어떻게 조정되나요?

난이도 손잡이는 **① 제시 단어 수(n)와 ② 최장 사슬 길이대역**입니다. 단어가 많고 정답 사슬이
길수록, 따져봐야 할 경로의 가짓수가 폭증해 '최장'을 찾기 어려워집니다.

**단어 꾸러미(POOL)**: 손으로 넣은 목록이 아니라 **공개 한국어 명사 사전**을 사용합니다 —
`generation/korean_nouns.txt` (출처: [han-dle/pd-korean-noun-list-for-wordles](https://github.com/han-dle/pd-korean-noun-list-for-wordles),
**CC0-1.0 퍼블릭 도메인**)에서 2~4글자 명사 **약 3,091개**를 추출해 저장. 생성기는 이 파일을
로드하며, 한국어 의미는 전혀 모르고 **글자만 비교**(`words[i][-1] == words[j][0]`)해 연결합니다.

문제는 **구성(construction) 방식**으로 만듭니다. 먼저 전체 풀에서 목표 길이만큼의 **백본 사슬을
무작위로 하나 뽑고**, 거기에 **방해 단어**를 섞어 n개를 만든 뒤, 그 n개의 **실제 최장 사슬을
정확히 계산**해 정답으로 씁니다. (랜덤으로 n개만 뽑으면 백본이 흩어져 사슬이 거의 안 생기므로,
이렇게 백본을 심어 난이도를 통제합니다.)

```python
# generation/word_chain_ko.py
PARAMS = {
    'easy':   dict(n=8,  lo=3,  hi=4),   # 단어 8개,  최장 3~4
    'medium': dict(n=18, lo=9,  hi=12),  # 단어 18개, 최장 9~12
    'hard':   dict(n=30, lo=16, hi=22),  # 단어 30개, 최장 16~22
}
# 백본 길이=lo로 사슬을 심고 방해 단어로 n까지 채운 뒤, 실제 최장값이 [lo,hi]면 채택.
# answer_cap = num//3 으로 한 길이값이 파일을 지배하지 않게 분산.
```

더 어렵게 하려면 `n`(단어 수)과 길이대역을 키우면 됩니다(단어 수가 커지면 정확해 계산도
느려지니 n은 17 안팎까지 권장).

## 평가
- `_parse_answer`: `정답: N` → `N개` → 응답 끝 정수 순으로 추출.
- `_check_answer`: 정수 일치.

## Gemini 3 Flash 기준선 (n=30/난이도, temp 0.6, ~3091 단어 사전 풀)
| easy | medium | hard |
|---|---|---|
| ~90% | ~73% | ~43% |

- **탐색 문제라 frontier 모델도 포화되지 않습니다**(산술인 korean_units가 100%였던 것과 대조).
  오답은 거의 **±1**(최장을 1 놓치거나, 안 이어지는 단어를 잘못 이어 1 부풀림).
- 난이도 손잡이(`n`·길이)를 키울수록 단조 하락: hard 기준 n17/9-12=63% → n24/12-16=50% →
  n30/16-22=**43%**. 산술 퍼즐과 정반대로 탐색이라 실제 어려워집니다. 작은 모델
  (Qwen 0.6B~8B)에서는 더 가파를 것으로 기대.

> 참고: 이 퍼즐을 PR/배포할 때는 생성기가 의존하는 `generation/korean_nouns.txt`(CC0)도 함께
> 포함해야 합니다.
