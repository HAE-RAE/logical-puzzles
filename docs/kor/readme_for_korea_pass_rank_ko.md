# 고개 높이 순위 추론 (korea_pass_rank_ko) — 난이도 설계 문서

- 생성기: `generation/korea_pass_rank_ko.py`
- 평가기: `evaluation/evaluators/korea_pass_rank_ko.py`
- 데이터: `data/jsonl/korea_pass_rank_ko_{easy,medium,hard}.jsonl` (생성기 출력)
- 태스크명: `korea_pass_rank_ko`, `..._easy`, `..._medium`, `..._hard`
- 보조 데이터: `generation/korea_passes.csv` (GeoNames KR PASS, **CC-BY 4.0**, 고개 한글명·고도·별칭수)

## 이 게임은 무엇인가요? (자세한 설명)

`korea_peak_rank`(산 높이)와 **완전히 같은 방식**을, 한국 고유 지형인 **고개(령·재·치·고개)**에
적용한 퍼즐입니다. 대관령·한계령처럼 이름난 고개도 있지만, 무수재·접치·송두재처럼 **대부분은
이름도 높이도 거의 알려지지 않은** 고개들이죠.

푸는 과정은 두 단계입니다.

**1단계 — 단서로 '탈락자' 골라내기 (추론).** "장재는 굴재보다 높다" 같은 단서가 있으면, **굴재는
'가장 높은 고개'가 될 수 없습니다** — 자기보다 높은 고개가 이미 있으니까요. 이렇게 단서에서 "누구보다
낮다"고 한 번이라도 적힌 고개를 모두 후보에서 지웁니다. 높이를 몰라도 **단서만 논리로 따져서** 하는
일이고, 이게 퍼즐(추론) 요소입니다.

**2단계 — 남은 후보 중 진짜 1등 고르기 (지식).** 지우고 나면 "한 번도 낮다는 말이 없는 고개" 몇 개가
남습니다. 단서는 이들끼리는 순위를 안 알려주므로, 진짜 가장 높은 고개를 고르려면 **실제 고개 높이
지식**이 필요합니다. (게다가 단서에서 많이 이긴 고개가 최고처럼 보여도, 단서에 한 번도 안 나온 고개가
더 높을 수 있는 **함정**이 있습니다.)

산 버전과 비교하면 **전반적으로 더 어렵습니다.** 사람들이 산 높이는 어느 정도 알아도, **고개 높이는
훨씬 덜 알려져** 있기 때문입니다(대관령이 832m인지 등). 그래서 잘 알려진 고개로 낸 easy조차 100%가
아니라 ~83%입니다.

## 문제 개념
- 고개 이름 목록 + "A는 B보다 높다" 단서 제시(고도 수치는 미제공). 가장 높은 고개를 묻는다.
- **정답 형식**: `정답: <고개 이름>`.
- **정답 산출**: 보기 고개들의 실제 고도(데이터)로 최고를 결정 → 단서는 그 최고보다 낮은 고개를
  "졌다"고 적어 후보를 줄여 주는 역할.

## 난이도는 어떻게 조정되나요?

핵심 손잡이는 **후보 고개의 '외짐 정도'**(GeoNames 별칭 수 nalt = 유명도 proxy)입니다. 후보 수(c)만
늘리면 곧 복권(랜덤)이 되므로, **외짐 tier**로 가릅니다.

```python
# generation/korea_pass_rank_ko.py
KNOWN   = nalt >= 6  또는 유명고개   # 잘 알려짐
MID     = nalt 4~5
OBSCURE = nalt <= 3                # 별칭 최소 = 가장 외짐
PARAMS = {
    'easy':   dict(n=5,  c=2, pool='KNOWN'),    # 잘 알려진 고개, 후보 2 → ~83%
    'medium': dict(n=8,  c=4, pool='MID'),      # 중간, 후보 4       → ~50%
    'hard':   dict(n=12, c=6, pool='OBSCURE'),  # 가장 외짐, 후보 6  → ~30%
}
# 신뢰성: 정답을 가르는 1·2위 고도차 ≥120m, 후보/탈락 경계 ≥50m. (고개는 표고차가 작아 120m)
```

- **gold 신뢰성**: 고도는 GeoNames dem(근사)이라 1·2위 차를 ≥120m로 둬 순서를 신뢰한다. "어려움
  (복권)"은 OK지만 "gold 오류"는 금지.
- hard의 후보 수(c)를 더 키우면 정답률이 랜덤(1/c)에 수렴하니, c=6(랜덤 ~17%보다 위)에서 멈췄다.

## 평가
- `_parse_answer`: `정답:` 뒤(또는 마지막 줄)의 고개 이름 추출.
- `_check_answer`: 정답 이름이 예측에 포함되면 정답(공백 정규화).
