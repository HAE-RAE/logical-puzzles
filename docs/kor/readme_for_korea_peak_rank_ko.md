# 산 높이 순위 추론 (korea_peak_rank_ko) — 난이도 설계 문서

- 생성기: `generation/korea_peak_rank_ko.py`
- 평가기: `evaluation/evaluators/korea_peak_rank_ko.py`
- 데이터: `data/jsonl/korea_peak_rank_ko_{easy,medium,hard}.jsonl` (생성기 출력)
- 태스크명: `korea_peak_rank_ko`, `..._easy`, `..._medium`, `..._hard`
- 보조 데이터: `generation/korea_mountains.csv` (GeoNames KR T/MT·PK, **CC-BY 4.0**, 한글명·고도·별칭수)

## 이 게임은 무엇인가요? (자세한 설명)

겉보기엔 "여러 산 중 가장 높은 산 고르기"라 그냥 상식 퀴즈처럼 보입니다. 만약 산 이름만 주고
"이 중 제일 높은 건?"이라고만 물으면, 그건 산 높이를 외웠는지 아닌지를 보는 **지식 퀴즈**일 뿐
퍼즐이 아닙니다.

그런데 이 문제에는 **'단서'**가 함께 붙어서 **논리적으로 생각해야 하는 단계**가 생깁니다. 예를
들면 "장성봉은 승기봉보다 높다", "장성봉은 가령산보다 높다" 같은 줄들이죠. 푸는 과정은 두
단계입니다.

**1단계 — 단서로 '탈락자' 골라내기 (추론).** 키 순서를 떠올려 봅시다. "철수가 영희보다 크다"는
말을 들으면, 영희는 그 반에서 '제일 큰 사람'이 될 수 없습니다 — 자기보다 큰 사람이 이미 있으니까요.
영희의 정확한 키를 몰라도 이 사실 하나로 후보에서 뺄 수 있습니다. 산도 똑같아서, 단서에서 "누구보다
낮다"고 한 번이라도 적힌 산은 모두 후보에서 지웁니다. 이건 산 높이를 전혀 몰라도 **단서만 논리로
따져서** 하는 일이고, 바로 이 부분이 퍼즐(추론) 요소입니다.

**2단계 — 남은 후보 중 진짜 1등 고르기 (지식).** 1단계로 지우면 "한 번도 낮다는 말이 없는 산"
몇 개가 남습니다. 단서는 이 후보들끼리는 누가 높은지 말해 주지 않으므로, 진짜 1등을 고르려면
이제 **실제 한국 산의 높이 지식**이 필요합니다.

**그리고 숨은 함정.** 단서에서 여러 번 "이긴" 산(여러 산보다 높다고 나온 산)이 1등처럼 보이지만,
진짜 1등은 **단서에 한 번도 안 나온 산**일 수 있습니다. 단서는 산들 관계의 *일부만* 알려주기
때문입니다(반에서 몇 명만 키를 재 봤다고, 안 재본 사람이 더 클 수 있는 것과 같죠). "많이 이긴 게
1등"이라는 성급한 결론을 경계하는 것 자체가 추론입니다.

정리하면, **"산 높이"라는 한국 고유 지식**에 **"단서로 후보를 추려내는 논리 추론"**을 얹은 하이브리드
퍼즐입니다. 쉬울수록 단서만으로 1등이 정해지고(추론), 어려울수록 단서로 다 못 좁혀 외진 산들의
높이 지식이 필요해집니다.

## 문제 개념
- 산 이름 목록 + "A는 B보다 높다" 단서 몇 개를 제시(고도 수치는 주지 않음). 가장 높은 산을 묻는다.
- **정답 형식**: `정답: <산 이름>` (보기 중 하나).
- **정답 산출**: 보기 산들의 실제 고도(데이터)로 최고를 결정 → 단서는 그 최고보다 낮은 산들을
  "졌다"고 적어 후보를 줄여 주는 역할(정답=참 최고는 불변).

## 난이도는 어떻게 조정되나요?

가장 강력한 손잡이는 **남는 후보 산들의 '외짐 정도'**입니다. 후보 수(c)만 늘리면 "1등만 맞히면
되니" Gemini 기준 ~67%에서 막히지만(plateau), **후보 산을 더 외지게(덜 알려지게)** 하면 모델이
높이를 몰라 급격히 어려워집니다.

외짐 척도로는 GeoNames의 **별칭 수(nalt)**를 유명도 proxy로 씁니다(별칭이 많을수록 잘 문서화·유명).

```python
# generation/korea_peak_rank_ko.py
# 외짐 tier (nalt = 별칭 수)
KNOWN   = nalt >= 6  또는 유명산   # 잘 알려짐
MID     = nalt 4~5
OBSCURE = nalt <= 3                # 별칭 최소 = 가장 외짐
PARAMS = {
    'easy':   dict(n=5,  c=2, pool='KNOWN'),    # 잘 알려진 산, 후보 2 → ~93%
    'medium': dict(n=8,  c=4, pool='MID'),      # 중간, 후보 4       → ~70%
    'hard':   dict(n=10, c=5, pool='OBSCURE'),  # 가장 외짐, 후보 5  → ~40%
}
# c = 단서로 안 좁혀지고 남는 후보 수. 신뢰성: 후보 1·2위 고도차 ≥150m, 후보/탈락 경계 ≥50m.
```

- **bimodal 주의**: Gemini의 산 지식은 "유명=다 앎 / 외짐=모름"으로 갈려 자연스러운 중간이 약함 →
  tier(KNOWN/MID/OBSCURE)로 외짐을 단계화해 중간 난이도를 만든다.
- **gold 신뢰성**: 고도는 GeoNames dem(근사)이라, 정답을 가르는 1·2위 고도차를 **≥150m**로 둬
  순서가 뒤집히지 않게 한다("어려움(복권)"은 OK, "gold 오류"는 금물).

## 평가
- `_parse_answer`: `정답:` 뒤(또는 마지막 줄)의 산 이름 추출.
- `_check_answer`: 정답 산 이름이 예측에 포함되면 정답(공백 정규화).

## Gemini 3 Flash 기준선 (n=30/난이도, temp 0.6)
| easy | medium | hard |
|---|---|---|
| ~93% | ~70% | ~40% |

- 단서 추론(후보 제거)은 Gemini가 잘하지만, hard의 **외진 산 5개 중 실제 최고**를 가르는 데서 크게
  떨어진다(40%). 후보 수가 아니라 **외짐 정도**가 난이도의 진짜 동인.
- 단서 없이 높이만 묻는 단순 지식 MCQ는 추론 단계가 없어 '퍼즐'로 보기 어렵다 — 이쪽(단서+지식)은
  논리 추론(후보 제거)이 더해져 logical-puzzles 성격에 맞는다.
